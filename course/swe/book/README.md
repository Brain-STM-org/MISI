# SWE Fundamentals: Textbook

This folder contains the written textbook content for the Software Engineering Fundamentals track. Each module is a self-contained chapter designed to be read alongside (or independently from) the slide presentations.

## Structure

The textbook is organized into three tiers that map to the delivery schedule:

### Tier 1: Foundations

| Chapter | Title | Description |
|---------|-------|-------------|
| [00](00-your-development-environment.md) | Your Development Environment | Terminal basics, VS Code, GUI vs CLI |
| [01](01-what-is-source-code.md) | What is Source Code? | Code as text, transformation to execution |
| [02](02-github-setup-guide.md) | GitHub Setup Guide | Account creation, 2FA, tools installation |
| [03](03-understanding-diffs.md) | Understanding Diffs | Reading diffs, the essential skill for AI-assisted work |
| [04](04-your-first-commit.md) | Your First Commit | The edit → stage → commit → push cycle |

### Tier 2: Collaboration & Quality

| Chapter | Title | Description |
|---------|-------|-------------|
| [05](05-branches-and-merging.md) | Branches and Merging | Parallel timelines, isolation, merging |
| [06](06-pull-requests.md) | Pull Requests | The collaboration workflow, code review |
| [07](07-testing-fundamentals.md) | Testing Fundamentals | Why test, types of tests, the testing mindset |
| [08](08-debugging-techniques.md) | Debugging Techniques | Reading errors, systematic debugging |
| [09](09-dependencies-packages.md) | Dependencies & Packages | npm, pip, evaluating dependencies |

### Tier 3: Professional Practices

| Chapter | Title | Description |
|---------|-------|-------------|
| [10](10-build-pipelines.md) | Build Pipelines | Automated verification, CI/CD concepts |
| [11](11-o11y-understanding-your-code.md) | O11y: Understanding Your Code | Logs, metrics, dashboards |
| [12](12-iterative-design.md) | Iterative Design | MVPs, shipping early, fighting perfectionism |
| [13](13-timeless-principles.md) | Timeless Principles | KISS, DRY, GIGO, YAGNI, Fail Fast |
| [14](14-communication.md) | Communication | READMEs, commit messages, comments |
| [15](15-ethics-in-software.md) | Ethics in Software | Bias, privacy, accessibility, dark patterns |

## Usage

- **Self-paced reading**: Students can read chapters independently
- **Reference material**: Use during project work to revisit concepts
- **Slide companion**: Each chapter corresponds to a module in `../deck/`
